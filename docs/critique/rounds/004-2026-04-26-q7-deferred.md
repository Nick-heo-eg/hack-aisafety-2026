## Q7 — Tool log 없는 tool은 어떻게?

### Decision

🟡→ **deferred** (사용자 결정).

### Why deferred (코드 작업 안 하는 이유)

1. **데모에 영향 0.** 우리 데모 시나리오(refund_api)는 자기 log 적는 toy tool.
   Q7은 *프로덕션 일반화 질문*이지 *데모를 깨는 질문*이 아님.
2. **답이 아키텍처/철학 영역.** 코드로 답하기보단 슬라이드/멘트로 처리하는 것이
   더 자연스러움 (3-channel 외부관찰: tool log / syscall capture / HTTP proxy).
3. **시간 우선순위.** 남은 시간은 발표 자료(슬라이드)와 P1 yellow 보강에 쓰는
   것이 더 큰 ROI.

### What this *doesn't* mean

- ❌ "답이 없다"가 아님. ADR-0006이 외부관찰을 *원리적으로* 다룸.
- ❌ "심사위원이 안 물을 것"이 아님. 물을 가능성 있음.

### What we'll say if asked

> "v0는 toy tool log로 *원리* 증명. 프로덕션은 3가지 외부관찰 채널 중 하나:
> (1) tool 자체 log (일부 SaaS), (2) syscall capture (strace), (3) HTTP proxy
> (mitmproxy). 어느 채널이든 *agent가 못 건드리는* 외부 관찰자가 핵심.
> ADR-0006 명시. 채널별 adapter는 v1+ 마일스톤."

### Strength rationale

- 🔴 red (round 001) → 🟡 deferred (round 004)
- *답은 있음*. *시연은 없음*. *시연 안 만들기로 결정*.
- "deferred"는 "yellow보다 약하지만 의식적 선택"의 표시.

### What would change this

다음 중 하나가 발생하면 round 005에서 deferred → in-progress 전환:
- 발표 D-1까지 시간 남음 → 3-channel 슬라이드만 추가
- 모의 발표에서 Q7이 실제 약점으로 드러남 → mitmproxy mini 데모 검토
