### Why this strength (정직한 자기평가)

**Strong:**
- Heuristic blind spot이 *측정값*으로 드러남 (0/4) — paraphrase 우회 *실제로
  존재함*을 자체 시연.
- 같은 H-002 코드 진입점에 layer만 추가. detector code clean.
- Real LLM judge 코드 *존재* + key 없을 때 *데모 안 죽음* 둘 다 보장.
- Demo 시연 자체가 답: "키워드만 아닙니다, 의미도 봅니다."

**Still imperfect:**
- 데모에 *실제* Gemini 호출 없음. MockJudge는 *더 넓은 패턴 매칭*일 뿐 — 본
  질적으로는 여전히 휴리스틱. **정직 명기**: "GeminiJudge available? False"
  데모 출력에 그대로 노출.
- MockJudge가 paraphrase 4/5 잡는 건 *우리가 그 paraphrase를 보고 짠 것*.
  out-of-distribution paraphrase는 못 잡을 가능성. 진짜 일반화는 LLM judge가
  필요.
- LLM judge prompt 자체가 휴리스틱 (`PROMPT_TEMPLATE`) — 미세 조정 안 함.

**판정: green.** 약점이 *데모 출력에 그대로 노출*되어 정직성 자체가 강도. 추격
당해도 깨지지 않음 — "GeminiJudge 코드 보여주세요" → 코드 있음, "키 받으면
어떻게 작동?" → 실행 코드 동일.
