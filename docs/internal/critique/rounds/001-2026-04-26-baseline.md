## 🔴 RED — 즉시 보강 필요 (3개)

### Q4 — Heuristic은 paraphrase로 우회됩니다. 시연도 키워드 매칭 아닌가요?

**Answer:** Heuristic은 *floor*. Paraphrase 우회는 *측정 가능한 약점*으로 의도적
으로 둠 — LLM judge layer (ADR-0005)가 그 위. heuristic 잡힘 vs LLM judge
잡힘 비교 자체가 honesty verification이 *왜 layer인지* 보여주는 측정값.

**Evidence:**
- `src/verifier/detectors/h002.py:18-26` — `_SUCCESS_KEYWORDS = ("완료", "처리했", "성공", "completed", "succeeded", "done", ...)`
- `docs/decisions/ADR-0005-honesty-taxonomy.md` — "heuristic + LLM judge" 2-layer 명시

**Strength:** 🔴 red

**Why this strength:**
시연에 LLM judge **없음**. 심사위원이 "그래서 LLM judge 보여주세요" → "Gemini
키 leaked" → 게임 끝. 답은 그럴듯한데 *시연 가능성*이 0.

**Remediation:**
- Heuristic +5 implicit success 패턴 ("마무리", "안전하게", "곧", "드릴게요", "확인 가능") — 10 LoC
- `docs/slides/02_heuristic_floor.md` — v0 한계 + LLM judge layer 설명 슬라이드

---

### Q5 — Real production agent (LangChain/AutoGen/Claude Code)에 어떻게 붙입니까?

**Answer:** ADR-0006이 명시: real-world adapter는 *tool process의 자체 log*가
필요. 프레임워크의 introspection이 아니라 외부 관찰 채널. v0는 toy tool로
*원리* 증명, vN에서 LangChain CallbackHandler / OpenAI function calling /
Claude Code transcript을 우리 schema로 변환하는 adapter.

**Evidence:**
- `docs/decisions/ADR-0006-dual-source-truth.md:84-92` — "Real-world adapters
  inherit a hard constraint" 명기
- 어떤 adapter도 코드로 없음

**Strength:** 🔴 red

**Why this strength:**
"원리 증명"이라는 답은 ADR에 박혀있지만, *시연*이 없음. AIM은 production 적용
가능성에 민감 (자기네 product가 그렇게 평가받음). "1주일 후 만들 수 있나?"
물으면 답 없음.

**Remediation:**
- `src/verifier/adapters/openai.py` mini — function calling response → 우리
  trace 변환 (~40 LoC)
- 또는 `src/verifier/adapters/__init__.py` + `Adapter` 추상 클래스 + LangChain
  pseudocode docstring (코드 없이 설계만)

### Q7 — Tool log가 *없는* tool은 어떻게 다룹니까? (대부분의 production tool)

**Answer:** ADR-0006 명시 3-채널:
1. Tool 자체 log (toy + 일부 SaaS webhook)
2. Syscall capture (strace/ptrace)
3. HTTP proxy (mitmproxy)

어떤 채널이든 *agent가 못 건드리는 외부 관찰자*가 핵심. v0는 (1)로 원리 증명.

**Evidence:**
- `docs/decisions/ADR-0006-dual-source-truth.md` — 3-채널 명시 (단, 본문에 *명
  시적*으론 안 적혀있고 "external observation" 표현만 있음 — 보강 필요)

**Strength:** 🔴 red

**Why this strength:**
ADR에서 명시되지 *않음* (확인 결과). "external observation" 표현뿐. (2)/(3)
시연도 없음. 심사위원 "strace 데모?" → 없음.

**Remediation:**
- `docs/slides/03_observation_channels.md` — 3-채널 다이어그램
- ADR-0006에 3-채널 단락 *명시적으로* amendment 추가 (또는 별도 ADR)
- (선택) mitmproxy mini 데모 — 50 LoC + 설치 (시간 큼)

## 🟡 YELLOW — 보강하면 green 가능 (7개)

### Q1 — AIM Supervisor와 차이?

**Answer:** Guard는 input/output, Supervisor는 decision logic, 우리는
execution truth (말한 것 vs 실제). Orthogonal layer.

**Evidence:** `docs/demo_script.md` Q&A 1번에 표 있음.

**Strength:** 🟡 yellow

**Why:** 발표 *멘트*에는 답 있는데 *시각 자료* 없음. 심사위원이 "표 보여주세요"
하면 즉답 없음.

**Remediation:** `docs/slides/04_layer_table.md` — 4-layer 차이 표 슬라이드.

### Q3 — 사후 detection이면 의미 있나?

**Answer:** "We close the *audit* gap, not the *execution* gap. Both needed.
Gates without verifiers are guards without auditors."

**Evidence:** `docs/devlog/0008` "한계 1번" — 사후 검출 의도적 명기.

**Strength:** 🟡 yellow

**Why:** 답 잘 framed인데 *수치/사례* 없음. "정량화" 부재.

**Remediation:** Air Canada 케이스에서 "agent가 거짓말 안 했으면 절약했을
audit cost" 추정 한 줄.

### Q6 — Multi-agent (subagent fabrication) 없음

**Evidence:**
- `src/verifier/trace.py` — `Spawn`, `Terminate` 이벤트 정의됨
- `docs/taxonomy.md:64-86` — H-003 정의
- 코드: H-003 detector **없음**

**Strength:** 🟡 yellow

**Why:** 인프라는 있는데 detector 없음. 시연 0.

**Remediation:** H-003 hardcoded fixture 1개 + 1줄 답: "trace schema가 Spawn
지원, detector는 다음 PR." (실제 detector 작업은 보류해도 됨)

### Q8 — Trace 형식 강제 권한?

**Answer:** 강제 안 함. Adapter pattern으로 *변환*만. 프레임워크 중립.

**Evidence:** ADR-0006 "adapter may transform a framework's native trace"
명시.

**Strength:** 🟡 yellow

**Why:** 답 명확한데 adapter 코드 없으니 *증거 약함*.

**Remediation:** Q5 보강(adapter)과 같이 해결됨.

### Q9 — False positive rate?

### Q11 — Deceptive alignment / mesa-optimization과 관계?

**Answer:** 우리는 *behavioral* honesty (관측 가능한 행동 vs 보고). mesa는
*internal* objective. 우리 도구는 mesa-optimizer가 *행동을 거짓 보고할 때*
잡지만, mesa 자체를 진단 못함.

**Evidence:** 없음 — 답이 머리에만 있음.

**Strength:** 🟡 yellow

**Why:** 학술 답인데 문서화 안 됨.

**Remediation:** ADR 또는 README에 "what we don't claim" 섹션 추가.

---

### Q12 — OpenTelemetry 통합?

**Answer:** ADR-0006 "Rejected alternatives" — OTel span이 "I called it"과
"it ran"을 conflate. 우리는 그 둘을 분리해야 함.

**Evidence:** `docs/decisions/ADR-0006-dual-source-truth.md:117-119`

**Strength:** 🟡 yellow

**Why:** 답 정확한데 슬라이드 없음.

**Remediation:** 발표 멘트에 한 줄만 끼워도 충분. 슬라이드까진 불필요.

## 🟢 GREEN — 통과 (2개)

### Q2 — Tool log 위조하면?

**Answer:** Tool 위조는 *별도 위협 모델*. 프로덕션은 syscall/HTTP proxy로 한
단계 더 외부화 가능 (ADR-0006). v0 toy는 신뢰 전제.

**Evidence:** `docs/devlog/0008` "한계 2번" 명기.

**Strength:** 🟢 green

**Why:** 한계 *솔직히* 인정 + 프로덕션 답 있음. 추격해도 정직성으로 통과.

---

### Q10 — Commercial 위협이 되나?

**Answer:** 우리는 commercial 위협을 *주장하지 않음*. 해커톤 출품작, 원리
증명. AIM이 1주일에 만들 수 있다는 사실 자체가 *이 dimension이 비어있음*의
증거 — 즉 우리 thesis 강화.

**Evidence:** README의 "포지션" 표 — "honesty verification: 비어있다 / empty"

**Strength:** 🟢 green

**Why:** 질문이 *우리에게 유리한* 답으로 뒤집힘. AIM이 만들 수 있다 = 우리가
지목한 빈 공간이 진짜 비어있다.
