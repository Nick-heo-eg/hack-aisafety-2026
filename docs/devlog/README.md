# Devlog

해커톤 진행 기록. 커밋이 *무엇*을 말하면, devlog는 *왜/어떻게*를 말한다.

## 규칙

- **Append-only.** 과거 엔트리 수정 금지. 정정은 새 엔트리에서.
- **한 엔트리 = 한 작업 세션.** 시간 박스, 작업량 박스 아님.
- **결정의 근거를 적는다.** 코드 변화는 git이 안다.
- **실패/우회/막힌 곳도 기록.** 안 그러면 같은 길 또 감.
- 파일명: `NNNN-YYYY-MM-DD-slug.md`

## 엔트리 형식 (가벼운 버전)

```markdown
# NNNN — Title (YYYY-MM-DD)

## What
무엇을 했나 (3~5줄)

## Why
왜 그렇게 했나 / 무엇을 버렸나

## Snags
막힌 곳, 의외였던 것, 실패

## Next
다음 한 발
```

섹션은 비어있어도 됨. 없는 섹션은 지운다.
