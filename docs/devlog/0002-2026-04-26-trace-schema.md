# 0002 — Trace schema (2026-04-26)

## What

- ADR-0002: trace는 JSONL + 닫힌 이벤트 union (7개 타입).
- `src/verifier/trace.py`: Pydantic v2 모델, `load_jsonl`(invariant
  체크 포함) / `iter_jsonl`(스트리밍) / `dump_jsonl`.
- F-001 시나리오를 사람이 읽을 수 있는 fixture JSONL로 박아둠 —
  taxonomy 문서가 *살아있는* 예제로 연결됨.
- 9개 테스트: 7개 이벤트 타입 파싱, 알 수 없는 타입 거부, extra 필드
  거부, fixture round-trip, step 역행 거부, 중복 terminate 거부, 빈
  줄 무시, 스트리밍 vs invariant 분리 검증.

## Why

- **닫힌 union vs free-form `metadata` dict.** 가장 큰 결정.
  free-form은 시작이 빠르지만 detector가 전부 방어적 코드가 된다 —
  `if "status" in ev.metadata` 같은 게 모든 detector에 퍼진다. 닫힌
  union은 *adapter 작성자*에게 비용을 떠넘기지만 detector를 선언적으로
  유지한다. detector는 우리가 많이 쓸 거고, adapter는 한 프레임워크당
  한 번 쓴다.
- **`step`을 monotonic int로 따로.** ts만으로는 같은 ms 내 순서가
  비결정적. F-003 (retry loop) 같은 패턴은 *순서*에만 의존한다.
- **invariant 체크를 `load_jsonl`에만, `iter_jsonl`은 패스.** 큰 trace
  스트리밍 처리할 때 메모리 다 안 잡고 싶음. 두 모드 분리하는 게 한
  함수에 플래그 박는 것보다 의도가 명확.
- **fixture를 taxonomy F-ID로 명명** (`f001_silent_fallback.jsonl`).
  나중에 detector 테스트할 때 fixture가 바로 정답지가 됨.

## Snags

- Pydantic v2의 discriminated union은 `Annotated[Union[...], Field(
  discriminator="type")]` 형태가 필요. v1의 `Field(..., discriminator=)`
  와 syntax 다름. 한 번 헷갈렸지만 문서 안 보고 fix됨.
- `extra="forbid"`를 base에 박았더니 v0 fixture에 무심코 필드 추가하면
  바로 깨짐 — 의도된 친구. 나중에 trace 확장할 때 명시적 결정 강제.
- `pytest.raises(Exception)` 두 군데 — pydantic이 `ValidationError`
  던지지만 import 더 깔끔하게 하려고 광의로 잡음. `noqa: B017` 달았음.
  나중에 detector 안정화되면 좁히기.
