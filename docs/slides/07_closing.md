## 🔑 closing이 하는 일

1. **다이어그램 재등장** — 처음에 본 시청자가 *다시 보면서* 모든 컷이 이걸로 수렴함을 깨달음
2. **"assume vs verify"** — 우리 thesis 한 번 더 박음
3. **"Two writers. One reader."** — 기억에 남을 한 줄
4. **GitHub URL + 43 tests** — *증거*는 GitHub에. 영상 끝나면 거기로 가게.

---

## 디자인

- Stage 2의 "We verify."는 *유일하게 강조 박스*. 다른 단어보다 *시각적 무게* 더.
- GitHub URL은 *작게*. 강조 아님. *추가 정보* 위치.
- 음악: hook과 같은 drone + 마지막 5초 약간 fade-out
- 컷 전환: closing 끝에 *완전 검정 fade-out* (0.5초)

## ⚠️ 절대 하지 말 것

- "감사합니다" / "Thank you" — 영상에 박지 말 것. 메시지 약화.
- 팀 사진 / 이름 — 시청자가 기억할 건 *thesis 한 줄*이지 우리가 누군지 아님
- "Q&A는 이메일로" — submission이 영상 + GitHub뿐임. Q&A 채널 없음.
- 다음 단계 / 로드맵 — 우리는 *완성된 것*을 보여줌, *계획*을 팔지 않음
- 다이어그램에 새 라벨 추가 — Stage 1은 02와 *완전히 동일*해야 함 (재인식)
