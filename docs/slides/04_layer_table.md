# Slide 4 — 4-Layer Table (Q1 답)

> **목적:** "AIM Supervisor와 뭐가 다르냐" 즉답.
> **위치:** 데모 직후. 심사위원 머릿속에 *우리 자리*를 박음.
> **시간:** 5초 시각, 5초 멘트.

---

## 🎬 시각 (이걸 띄움)

```
┌──────────────────────┬──────────────────┬──────────────────────────┐
│ Layer                │ 본다              │ 가정                     │
├──────────────────────┼──────────────────┼──────────────────────────┤
│ Guardrail            │ input / output   │ agent의 출력은 신뢰 가능   │
│  (AIM Guard)         │                  │                          │
├──────────────────────┼──────────────────┼──────────────────────────┤
│ Gate                 │ execution 차단    │ agent의 의도는 신뢰 가능   │
│  (AIM Starfort)      │                  │                          │
├──────────────────────┼──────────────────┼──────────────────────────┤
│ Supervisor           │ decision logic   │ agent의 추론은 신뢰 가능   │
│  (AIM Supervisor)    │                  │                          │
├══════════════════════┼══════════════════┼══════════════════════════┤
│ Honesty Verifier     │ 말한 것 vs       │ ❌ 아무것도 신뢰 안 함     │
│  (우리)               │  실제 일어난 것    │  ── 검증 대상              │
└──────────────────────┴──────────────────┴──────────────────────────┘
```

---

## 🎤 5초 멘트

> "위 세 layer는 모두 *agent를 신뢰*합니다.
> 우리는 *그 신뢰 자체*를 검증합니다.
> Orthogonal — 같은 dimension에서 경쟁하지 않습니다."

---

## 🔑 핵심 포인트

- **Orthogonal (직교)** 단어가 핵심. "위/아래/대체"가 아님.
- AIM 3개 product 모두 *언급*. 인정 + 차별화 동시.
- "검증 대상" = 우리 도구의 *주체*. 다른 layer는 가정으로 사용하는 것을 *우리만 검증함*.

---

## ⚠️ 절대 하지 말 것

- ❌ "AIM 제품의 한계입니다" — 정면 도전 framing 금지
- ❌ "Supervisor가 못 잡는 것" — 그들 product 깎는 표현 금지
- ✅ "다른 dimension" — 보완도 비판도 아닌 *직교*

---

## 후속 질문 대비

**Q. "Supervisor도 자기보고 검증 일부 합니다."**
> "맞습니다. Supervisor는 *agent의 decision*을 봅니다. 우리는 *agent의 행동*을
> 봅니다. 결정은 trace에 자기보고로 들어오고, 행동은 외부 관찰(tool log)에서
> 들어옵니다 — *두 source*. 이 분리가 우리 thesis."

**Q. "그러면 Supervisor + 우리 같이 쓰면 되나?"**
> "정확히 그게 우리 포지션입니다. 같이 쓰는 *layer*. 경쟁 product 아님."
