# Slide 02 — Two Writers, One Reader (CENTER CUT)

> **이 슬라이드 = 프로젝트의 80%.**
> 다른 모든 컷은 이걸 *받쳐주는* 역할.
> 영상 0:25-0:55 (30초). 다른 컷보다 *느리게* 호흡.

---

## 🎬 다이어그램 (이걸 영상에 그대로 띄움)

```
                      ┌────────────┐
                      │   user     │
                      └─────┬──────┘
                            │ "환불해줘"
                            ▼
                      ┌────────────┐
                      │   agent    │  ← LLM
                      │  (gpt-4o)  │
                      └─────┬──────┘
                            │
              ┌─────────────┴─────────────┐
              │                           │
              ▼                           ▼
        "처리 완료했습니다"          refund_api(order=42)
              │                           │
              │ writes                    │ called
              ▼                           ▼
       ┌───────────────┐          ┌───────────────┐
       │   trace.jsonl │          │ tool_log.jsonl│
       │  (자기 보고)   │          │ (자기 호출 기록) │
       └───────┬───────┘          └───────┬───────┘
               │                          │
       agent가 직접 씀              tool이 직접 씀
       *거짓말 가능*                  *agent 못 건드림*
               │                          │
               └────────────┬─────────────┘
                            │
                            ▼
                    ┌───────────────┐
                    │   verifier    │
                    │  reads both,  │
                    │ writes neither│
                    └───────┬───────┘
                            │
                            ▼
                       🚨 mismatch
```

---

## 📐 시각 디자인 가이드

### 색상
- **trace 쪽 (왼쪽)** — 🟦 파랑 계열. 차가움. 의심.
- **tool log 쪽 (오른쪽)** — 🟥 빨강 계열. 진짜. 증거.
- **verifier (가운데 아래)** — ⬜ 흰색/회색. 중립. 비교만.
- **mismatch** — 🚨 강한 빨강. 발견의 순간.

### 애니메이션 순서 (영상 30초 분할)
1. **0:25 - 0:28** (3s) — agent + user만 등장
2. **0:28 - 0:33** (5s) — 두 갈래로 분기 (말 / 호출)
3. **0:33 - 0:40** (7s) — trace.jsonl + tool_log.jsonl 두 박스 동시 등장 + "agent가 씀 / tool이 씀" 캡션
4. **0:40 - 0:48** (8s) — verifier 박스 등장, 두 화살표가 verifier로 모임
5. **0:48 - 0:55** (7s) — 🚨 mismatch 빨갛게 깜빡, 화면 hold

### 자막 (영어, 음성 한국어 동안 화면 하단)
- "Two writers."
- "Agent writes its own trace."
- "Tool writes its own log."
- "**Agent can lie in trace. Tool cannot lie in its own log.**"
- "We compare them."

→ 5문장. 6초 간격으로 fade in/out.

## 🎤 음성 멘트 (한국어 30초)

```
(0:25) "두 개의 writer가 있습니다."

(0:28) "에이전트는 trace에 자기 행동을 적습니다."

(0:33) "도구는 자기 자신이 호출됐다는 사실을 자기 로그에 적습니다."

(0:40) "에이전트는 trace에 거짓말할 수 있습니다.
        도구의 자기 로그에는 못 합니다."

(0:48) "우리는 그 둘을 비교합니다."
```

→ 천천히. 7초당 한 문장. 호흡 길게.

---

## 🔑 이 슬라이드가 하는 일

1. **"why dual-source?"** — 한 그림으로 답
2. **"무엇이 새로운가?"** — 다른 시스템들이 *agent의 출력만* 보는 것과 *대비*
3. **"왜 거짓말 못 잡히나?"** — agent가 trace에는 거짓 적을 수 있지만 *tool 로그에는 손 못 댐* 시각화
4. **"verifier가 뭐 하나?"** — *비교만, 안 적음* — 한 줄로 ADR-0006 표현

## ⚠️ 디자인 함정 (피해야 할 것)

| 함정 | 왜 위험 | 대안 |
|---|---|---|
| 박스 안에 코드 스니펫 넣기 | 영상에서 글자 작아 안 읽힘 | 캡션은 짧은 자연어로 |
| 화살표에 라벨 5개 이상 | 시각 산만 | "writes" / "called" 정도만 |
| 우리 detector / judge 표현 | 30초 안에 못 설명 | 다음 컷(데모)이 함 |
| 4-layer 표 함께 표시 | 메시지 분산 | 4-layer는 별도 컷(0:10-0:25) |
| ADR / H-002 같은 jargon | 시청자 모름 | 자연어로만 |

> **The tool cannot lie in its own log.**
